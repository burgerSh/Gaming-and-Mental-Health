# 🎮 Gaming Addiction Risk Predictor (ระบบประเมินความเสี่ยงการติดเกมด้วย AI)

[![Streamlit App](https://gaming-and-mental-health-jpc9dqnyyepewj9jknamqw.streamlit.app/)

แอปพลิเคชัน Machine Learning สำหรับประเมินความเสี่ยงในการติดเกมเบื้องต้น พัฒนาขึ้นเพื่อช่วยให้ผู้ใช้งาน (เกมเมอร์ หรือผู้ปกครอง) สามารถตระหนักรู้ถึงพฤติกรรมการเล่นเกมของตนเองว่าอยู่ในระดับที่เหมาะสม หรือมีความเสี่ยงที่ควรปรับเปลี่ยนพฤติกรรม

> **Disclaimer:** ผลลัพธ์จากการทำนายไม่ใช่การวินิจฉัยทางการแพทย์ หากผู้ใช้งานมีความกังวลเกี่ยวกับสุขภาพจิต ระบบจะแนะนำให้ปรึกษาแพทย์หรือผู้เชี่ยวชาญทันที

---

## 📌 1. การนิยามปัญหา (Problem Statement)
ในยุคดิจิทัล การเล่นเกมเป็นกิจกรรมยามว่างที่ได้รับความนิยมสูง แต่การเล่นเกมที่ขาดการควบคุมสามารถนำไปสู่ **"ภาวะติดเกม" (Gaming Disorder)** ซึ่งส่งผลกระทบต่อสุขภาพร่างกาย จิตใจ และการเข้าสังคม โปรเจคนี้จึงถูกสร้างขึ้นเพื่อเป็นเครื่องมือคัดกรองเบื้องต้น (Screening Tool) โดยใช้ข้อมูลพฤติกรรม อาการทางร่างกาย และจิตใจ มาวิเคราะห์หาระดับความเสี่ยง เพื่อให้ผู้ใช้รู้เท่าทันและปรับเปลี่ยนพฤติกรรมได้ทันท่วงที

---

## 📊 2. ข้อมูลที่ใช้ (Dataset & Features)
ชุดข้อมูลที่ใช้รวบรวมพฤติกรรมของผู้เล่นเกมหลากหลายกลุ่ม โดยมีตัวแปร (Features) ที่สำคัญดังนี้:
* **ข้อมูลทั่วไป:** อายุ (Age), เพศ (Gender)
* **พฤติกรรมการเล่น:** แนวเกม (Game Genre), แพลตฟอร์ม (Platform), ชั่วโมงการเล่นต่อวัน (Daily Hours), ค่าใช้จ่ายในเกม (Monthly Spending)
* **ตัวชี้วัดทางสังคมและจิตวิทยา:** คะแนนการแยกตัวจากสังคม (Social Isolation Score)
* **อาการทางร่างกายและจิตใจ (Red Flags):** * หงุดหงิด/กระวนกระวายเมื่อไม่ได้เล่น (Withdrawal Symptoms)
  * เบื่อหน่ายกิจกรรมอื่นที่เคยชอบ (Loss of Other Interests)
  * ปวดตา / ตาล้า (Eye Strain)
  * ปวดหลัง / ปวดคอ (Back & Neck Pain)

---

## 🧠 3. การพัฒนาโมเดลและการประเมินผล (Model Development)

### อัลกอริทึมที่เลือกใช้: **Random Forest Classifier**
* **เหตุผลที่เลือก:** Random Forest ทนทานต่อข้อมูลที่มีความหลากหลาย (ทั้งตัวเลขและหมวดหมู่) ป้องกันการ Overfitting ได้ดี และที่สำคัญคือสามารถดึงค่า **Feature Importance** ออกมาอธิบายได้ว่าปัจจัยใดมีผลต่อการติดเกมมากที่สุด ซึ่งตอบโจทย์การนำไปสื่อสารให้ผู้ใช้งานเข้าใจ
* **การจัดการข้อมูล (Pipeline):** มีการทำ One-Hot Encoding สำหรับข้อมูลหมวดหมู่ (Categorical Data) และจัดรูปแบบคอลัมน์ให้อยู่ในโครงสร้างเดียวกับตอน Train

### ประสิทธิภาพของโมเดล (Model Evaluation)
เนื่องจากปัญหานี้เกี่ยวกับการคัดกรองความเสี่ยงทางสุขภาพ โมเดลจึงให้ความสำคัญกับค่า **Recall** และ **F1-Score** เพื่อป้องกันไม่ให้ AI ทำนายผู้ที่มีความเสี่ยงสูงพลาด (False Negatives)
* **Accuracy (ความแม่นยำรวม):** `88.44%`
* **Recall (ความครอบคลุมกลุ่มเสี่ยง):** `87.00%`
* **F1-Score (ค่าเฉลี่ยประสิทธิภาพสมดุล):** `88.00%`

> **⚙️ Rule-based Override (การแปลผลเชิงปฏิบัติ):** นอกจากการใช้ ML แล้ว ระบบได้เสริมกฎเกณฑ์พิเศษทางการแพทย์เข้าไป หากผู้ใช้มีอาการรุนแรง (เช่น หงุดหงิดเมื่อไม่ได้เล่น หรือ เบื่อหน่ายกิจกรรมอื่นในชีวิตจริง) ระบบจะบังคับให้ผลลัพธ์เป็น **"เสี่ยงสูง (Severe)"** ทันที เพื่อความปลอดภัยของผู้ใช้งาน

## 🚀 4. การใช้งานแอปพลิเคชัน (Deployed Web App)
แอปพลิเคชันถูก Deploy ผ่าน **Streamlit** โดยออกแบบ UI ให้ใช้งานง่าย แบ่งออกเป็น 2 ส่วนหลัก:
1. **📝 ทำแบบประเมินความเสี่ยง:** ฟอร์มรับข้อมูลที่ใช้งานง่าย (Input Validation ครบถ้วน) พร้อมการแสดงผลลัพธ์ที่แบ่งระดับชัดเจน (Low, Moderate, Severe)
2. **🧠 เกี่ยวกับโมเดล AI (Bonus Feature):** นำเสนอเบื้องหลังการทำงานของโมเดล พร้อมกราฟแท่งแบบ Interactive (สร้างด้วยไลบรารี `Altair`) แสดง Feature Importance ให้ผู้ใช้เข้าใจว่า AI ใช้น้ำหนักจากปัจจัยใดในการตัดสิน
